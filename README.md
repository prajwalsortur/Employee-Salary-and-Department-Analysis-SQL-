# 📊 Employee Salary & Department Analysis

## 📌 Project Overview
The **Employee Salary & Department Analysis** project focuses on analyzing employee compensation data across multiple departments using structured relational database concepts.

This project demonstrates practical SQL knowledge applied to real-world HR analytics scenarios. It helps in understanding salary distribution, departmental salary comparisons, and compensation trends to support data-driven decision-making.

---

## 🎯 Project Objectives
- Analyze salary distribution across departments
- Identify highest and lowest paying departments
- Calculate average, minimum, and maximum salaries
- Compare department-wise salary trends
- Demonstrate strong SQL fundamentals in a business context

---

## 🛠️ Tools & Technologies Used
- **Database:** MySQL
- **SQL Client:** MySQL Workbench
- **Version Control:** Git & GitHub
- **Visualization (Optional):** Power BI / Tableau

---

## 🗂️ Database Structure
The project consists of two relational tables:

### 🔹 Employees Table
Stores employee personal details and salary information.

### 🔹 Departments Table
Contains department details and is linked to the Employees table using a foreign key relationship.

The database design follows:
- Primary Key constraints
- Foreign Key relationships
- Normalized schema design
- Referential integrity principles

---

## 📊 Key Analysis Performed
- Employee-to-department mapping using JOIN operations
- Department-wise average salary calculation
- Highest and lowest salary identification
- Aggregate analysis using GROUP BY
- Filtering results using HAVING clause
- Salary comparison across departments
